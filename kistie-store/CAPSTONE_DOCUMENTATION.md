## 1. Introduction
Kistie Store is a modern, full-stack fashion e-commerce platform for fashion retail. The project was motivated by the need for a professional, responsive, and easy-to-use online store for both customers and administrators. It aims to streamline inventory management, shopping, and admin operations.
## 2. Project Description
Kistie Store provides a seamless online shopping experience for the fashion industry. It solves the problem of managing inventory, orders, and customer experience for small-to-medium fashion retailers, especially in Uganda and Kenya. 
**Key Features:**
- Admin authentication and dashboard
- Product catalog with images, prices, and details
- Add, edit, and delete products (CRUD)
- Shopping cart and order placement
- RESTful API (Node.js/Express)
- SQLite database for persistent storage
- Responsive, modern UI (HTML, CSS, JS)
- CORS and security best practices
- Dual-currency support (UGX/KES)
- Size and quantity selection
- Interactive catalog/gallery experience
## 3. Data Entities
- **User**: Admins and customers (see backend/user table)
- **Product**: Name, description, price, images, sizes, currency (see backend/product table)
- **Order**: Items, quantities, user info, total price, currency
- **Cart**: Session-based or user-based cart items
## 4. Industry and Target Audience
**Industry:** Fashion e-commerce, retail technology  
**Target Audience:** Fashion shoppers in Uganda and Kenya, small-to-medium fashion retailers, and online store administrators.
## 5. Elevator Pitch
Kistie Store is a professional, modern e-commerce platform for fashion retail, offering a seamless shopping experience, robust admin tools, and support for multiple currencies and sizes. Its responsive design and interactive features make it ideal for both customers and store managers.
## 6. Technical Overview
- **Backend:** Node.js, Express, SQLite
- **Frontend:** HTML, CSS, JavaScript
- **Deployment:** Render.com
**Reasons for Technology Choices:**
- Node.js/Express for fast, scalable backend
- SQLite for lightweight, easy-to-deploy database
- HTML/CSS/JS for maximum compatibility and responsiveness
**Architecture Overview:**
- RESTful API backend
- Static frontend served via HTML/CSS/JS
- Database for persistent storage
